# SQL Server 備份與還原規範

本專案採用結構化的備份管理方式，讓開發與測試環境可以透過 `.bak` 檔快速重建，也可以把容器中已修改的資料庫重新備份回本機。

## 備份目錄結構

建議所有備份都放在 `backups/{database}/latest.bak`。`docker-compose.yml` 會把本機 `./backups` 掛載到容器內的 `/var/opt/mssql/backups`，因此容器內的備份與還原都會直接對應到本機檔案。

```text
backups/
├── gtmk/
│   ├── latest.bak       # 核心備份檔 (Entrypoint 自動掃描此名稱)
│   └── metadata.json    # 備份中繼資料
├── hccg/
│   ├── latest.bak
│   └── metadata.json
└── .gitkeep
```

## Metadata 規範

每個備份目錄下應包含一個 `metadata.json`，範例如下：

```json
{
  "database": "gtmk",
  "sqlserverVersion": "2022",
  "source": "production",
  "createdAt": "2026-04-23",
  "description": "支票批號異常重現資料"
}
```

`backup.sh` 產生的 `metadata.json` 會使用固定格式：

```json
{
  "database": "gtmk",
  "sqlserverVersion": "2022",
  "source": "backup.sh",
  "createdAt": "2026-06-08",
  "description": "Backup of gtmk database generated by backup.sh"
}
```

## 自動還原機制

當容器啟動時，`entrypoint.sh` 會自動執行以下邏輯：

1. **特定還原**：若環境變數 `RESTORE_DB` 有值（例如 `RESTORE_DB=gtmk`），則僅針對該資料庫進行還原。
2. **全量還原**：若 `RESTORE_DB` 為空，則自動掃描 `/var/opt/mssql/backups/*/latest.bak`，並依序還原。
3. **跳過機制**：若資料庫已存在於系統中，則會跳過還原流程，避免覆蓋既有資料。
4. **舊格式相容**：除了結構化目錄，也會掃描 `/var/opt/mssql/backups/*.bak`，以支援舊版扁平檔案格式。

### 手動觸發特定還原

在啟動時指定環境變數：

```bash
RESTORE_DB=hccg docker compose up
```

## 資料庫備份腳本

為了方便從現有容器中將修改後的資料庫備份出來，專案內建了 `backup.sh` 腳本，可自動將運行中的 SQL Server 資料庫備份回宿主機的 `backups/` 目錄。

### 前置條件

執行前請先確認：

1. `.env` 內有正確的 `SA_PASSWORD`，若有自訂登入帳號也要設定 `USER_NAME`。
2. `mssql-eris` 容器已啟動。
3. 容器內可使用 `sqlcmd` 連線到 SQL Server。

若未設定 `SA_PASSWORD`，腳本會改用互動式輸入密碼。

### 使用方式

1. **互動式選單備份**：
   不帶參數執行腳本，腳本將自動查詢容器中所有的使用者資料庫，並提供選單讓您選擇備份單一資料庫、全部備份或取消：

   ```bash
   ./backup.sh
   ```

2. **指定資料庫備份**：
   若已知資料庫名稱，可直接加在參數後方（例如備份 `gtmk`）：

   ```bash
   ./backup.sh gtmk
   ```

### 備份流程

1. 載入 `.env` 中的連線資訊。
2. 自動偵測目前使用 `podman` 或 `docker`。
3. 檢查 `mssql-eris` 容器是否正在執行。
4. 查詢 SQL Server 內所有使用者資料庫，排除 `master`、`tempdb`、`model`、`msdb`。
5. 依照參數或互動選單決定備份目標。
6. 建立 `backups/{database}` 目錄。
7. 若已存在 `latest.bak`，先歸檔為 `latest.YYYYMMDD_HHMMSS.bak`；若檔名仍衝突，會再加上 process id。
8. 在容器內執行 SQL Server `BACKUP DATABASE`，輸出到 `/var/opt/mssql/backups/{database}/latest.bak`。
9. 產生或更新 `metadata.json`。

### 備份特點

* **舊備份歸檔**：新備份會固定寫到 `latest.bak`，原本的 `latest.bak` 會先改名為時間戳檔案。
* **環境自動偵測**：自動檢測本機是使用 `podman` 或 `docker` 執行容器。
* **自動產生 Metadata**：備份完成後會自動產生或更新對應的 `metadata.json`，維持結構規範。
* **可接續還原流程**：產出的 `backups/{database}/latest.bak` 可被 `entrypoint.sh` 在下次啟動時自動掃描還原。

## 注意事項

* **SA 密碼**：備份與還原都需要足夠權限，請確保 `.env` 中的 `SA_PASSWORD` 正確。
* **資料庫已存在時不覆蓋**：自動還原時若目標資料庫已存在，`entrypoint.sh` 會跳過該資料庫。
* **邏輯檔名**：還原腳本會自動偵測 `.bak` 內的邏輯檔名，並對應到 `/var/opt/mssql/data` 下的 `.mdf` 與 `.ldf`。
* **大型資料庫耗時**：備份與還原都使用 `STATS = 10` 顯示進度，大型資料庫需要等待較久。
* **資料庫命名**：建議使用不含空白與特殊字元的資料庫名稱，避免 shell 參數與 SQL 指令解析問題。
